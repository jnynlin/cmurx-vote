1核心架構（解剖學）

系統分三層，資料由上往下單向流動：採集 → 儲存 → 分析。兩個獨立的儲存後端（Firebase 與 Google Sheet）分別承載「即時互動」與「成績結算」。

單向資料流原則：下游全部唯讀——分析工具只讀不寫，永不污染原始資料。唯一的寫入動作（關閉投票、刪除帳號）保留在教師端 zzzzzz.html（admin 認證）。

2計分生理（兩套循環系統）

系統內有兩套獨立的計分循環，在一個點交會。理解它們的分流與匯流，是理解整個評量設計的關鍵。

🩸 循環 A：個人成績

衡量「每位學生的課堂參與」。由 GAS 在教師同步時計算，寫入個人成績表。

項目分數
校準 calibration5
反思 rating10
作業 assignment15
展覽 gallery15
閱讀 notebook (秒讀<15s 罰)5–15
互動 slido (秒關<5s 罰)5–15
測驗 forms (秒填<20s 罰)10–25
協作貢獻 (每項3, 上限)0–9
組長加成 (rating×2, 上限)0–10
海報票選分 無上限0–N×?
非海報項上限 ≈ 119 分；海報票選分直接累加，無封頂。

🏆 循環 B：海報評選

衡量「海報作品的排名」。有兩個獨立來源，產出典禮排名，不直接計入個人成績（除了 B1 的票數回流到循環 A）。

B1 · 學生票選（人氣）

// 每位學生選 3 名，Borda 計分
第1名 +3　第2名 +2　第3名 +1
組票數 = Σ 全班給該組的分

B2 · 評審評分（學術）

// 3 評審 × 4 面向(1–5)
加權分 = 0.5×平均(選題,病理,視覺)
　　　　 + 0.5×整體印象
最終分 = 平均(A,B,C 加權分)
　（未送出的評審不計入平均）

⚡ 交會點：海報票選分（循環 B1 → 循環 A）

// 學生投票的組票數，回流成每位組員的個人成績
個人.poster_score = 組Borda票數　(整組相同, 無上限)
個人總分 = 課堂項(≤119) + poster_score

設計決策（刻意不設上限）：實測海報分範圍 63–169，個人成績差距達 106 分。經分析——只有「無上限」能讓第 1 名組的每位成員，在個人成績上都高於第 12 名組（差距 66 分、無重疊）。任何形式的封頂都會造成組別成績混疊，使票選失去鑑別意義。

3第三道視角：AI 綜合分析（旁路）

AI rubric 評估是一條旁路（不進成績、不進排名），純作為第三方參考，與評審、學生票交叉比對。同樣 50/50 加權，外加團隊合作加分。

AI 基礎分 = 0.5×平均(選題,病理,視覺) + 0.5×整體　// 同評審公式
團隊加分 = max(0, (團隊分−3) × 0.25)　// 上限 +0.5
三方分流的意義：學生票＝人氣／溝通；評審＝學術深度；AI＝一致性參考。三者衡量不同的東西——「不一致」本身就是最有價值的教學訊號（如 ATC：AI 第一、評審第十、學生第十二＝深度≠吸引力）。

4系統恆定（資料完整性機制）

票數恆定律

投出總分 = 投票人數 × 6
（每人恰好 3+2+1=6 分）

若總分 = N×6 → 無灌票、無覆蓋。這是結案的鐵證（master_console 即時驗算）。

解鎖閘門

投票卡僅在 gallery.done 後出現——所以「未投票」多為結構性（沒完成展覽，卡未解鎖），非冷漠。提高投票率的槓桿是「完成展覽」。

DEBUG 根除

真實學號格式 S11[1-4]003xxxx；測試帳號（S1101/S1102/S111000999）在所有分析入口 root 排除。
